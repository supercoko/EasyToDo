# 打包指南（Windows / Tauri 2）

本文档说明如何把当前项目打包成可分发安装包。

## 1. 前置环境

- Node.js（建议 LTS）
- `pnpm`
- Rust 工具链（`rustup + stable`）
- Windows 构建环境（MSVC Build Tools）

可先执行以下命令确认：

```powershell
node -v
pnpm -v
rustup toolchain list
```

如果 Rust 未配置默认工具链，执行：

```powershell
rustup default stable
```

## 2. 安装依赖

在项目根目录执行：

```powershell
pnpm install
```

## 3. 生成安装包（推荐）

执行：

```powershell
pnpm tauri build --bundles nsis
```

说明：

- 该命令会先执行前端构建（`pnpm build`）
- 再编译 Tauri/Rust 程序
- 最终产出 Windows 安装包（`.exe` 安装程序）

## 4. 产物位置

### 安装包（分发给用户）

```text
src-tauri/target/release/bundle/nsis/防遗忘待办_0.1.0_x64-setup.exe
```

> 文件名中的版本号会随 `src-tauri/tauri.conf.json` 的 `version` 变化。

### 可执行文件（免安装）

```text
src-tauri/target/release/focus-float-todo.exe
```

## 5. 常见问题

### 5.1 `rustup could not choose a version of cargo`

原因：没有默认 Rust 工具链。  
解决：

```powershell
rustup default stable
```

### 5.2 第一次打包较慢

原因：首次会下载并编译大量 Rust 依赖、可能下载 NSIS 组件。  
解决：属于正常现象，后续会明显变快。

### 5.3 修改应用版本号

编辑：

```text
src-tauri/tauri.conf.json
```

将 `version` 改为目标版本后重新执行打包命令。

## 6. 常用命令速查

```powershell
# 前端生产构建
pnpm build

# Tauri 默认打包
pnpm tauri build

# 仅打 Windows NSIS 安装包
pnpm tauri build --bundles nsis
```
